Gubbu.io System Architecture

Agentic AI System Design, Scalability, and Performance Optimizations

🤖 Agentic AI System Design

We are actively transitioning Gubbu.io into a Decision-as-a-Service (DaaS) platform using Agentic AI concepts.

  • Current State: A single orchestrator agent using rules, RAG, and few-shot prompting with low temperature (deterministic).
  • Future Multi-Agent System: We are planning specialized agents: Destination Advisor, WFA Advisor, Lifestyle Picks, and Booking Intent Classifier.
  • Shared Infrastructure: All agents share the same routing, compliance rules, and observability layers.

📈 Scalability & High Availability

The architecture is designed to handle high loads gracefully without bottlenecks.

  • Stateless Backend: A lightweight Node.js API server acts purely as an orchestration/routing layer, allowing infinite horizontal scaling.
  • Frontend Caching: Vite-powered React app delivered via CDN (Firebase Hosting) ensures high availability globally.
  • No Direct Bookings: The system is non-transactional, offloading the heavy lifting of payments and real-time inventory to affiliate partners.

🔒 Security & Guardrails

Security and hallucination prevention are built into the core AI interactions.

  • Strict Prompting: System rules forbid external platform names, fabricated dates, or real-time pricing claims.
  • RAG Grounding: Only verified blog content from Supabase is used as context. Low similarity chunks (< 0.7) are discarded.
  • Controlled Endpoints: Affiliate links are securely generated on the backend via /go/* redirects, preventing client-side manipulation.

⚡ Performance Offloading

Ensuring lightning-fast load times across the application.

  • Build-Time Processing: Markdown files are parsed via import.meta.glob at build time, converting to HTML before deployment.
  • Client-Side Routing: React Router enables instant page transitions without full page reloads.

🗄️ Database Scaling & Optimization

Optimized data retrieval leveraging vector search capabilities.

  • Supabase pgvector: RAG embeddings (text-embedding-3-small, 768 dimensions) are stored and matched efficiently inside Postgres.
  • Asynchronous Logging: Click logging and observability metrics are fired asynchronously, never blocking user requests.

🌐 Traffic Distribution & Communication

Clean separation of concerns for network requests.

  • API-First Approach: The UI communicates seamlessly with api.gubbu.io.
  • Affiliate Router: A centralized routing layer dynamically resolves rules (based on Country, City, Priority) before issuing a 302 redirect to distribute traffic to the optimal partner.
